Ⅲ · Appendix Glossary
Terms of the brief — units, standards, source fields, processing stages and quality flags.
Units & standards
mBGL
Unit
Metres Below Ground Level. The canonical vertical unit for groundwater depth — a positive number of metres beneath the surface at a site. All operator-native units resolve to water_level_mbgl once normalised.
WaterML 2.0
Standard
OGC standard for exchange of hydrological time-series. Anchors the framework's controlled vocabulary and record structure — every source field is mapped to a WaterML-compliant slot before release downstream.
Datum
Reference
The fixed vertical reference against which water-level measurements are expressed — commonly top-of-casing, ground level, or mean sea level. The connector stores one datum per site and applies the offset at ingestion.
Source fields
GW_depth_m
SWALIM
Native field used by SWALIM / FAO. Depth to groundwater in metres, relative to ground surface. Conceptually aligned with water_level_mbgl.
wt_below_casing
World Vision
Water-table depth measured from the top of the well casing. Requires a per-site casing-to-ground offset to normalise onto the common ground-referenced datum.
borehole_level
INGO 3
Vertical position inside the borehole reported by the operator's logger. Interpretation depends on the transducer mount and the operator's datum convention, so the adapter preserves the raw value alongside provenance before normalisation.
Processing
Adapter registry
Stage 1
Pluggable ingestion layer holding one adapter per operator. Each adapter handles extract, buffer, and retry against the source feed, preserving raw records and provenance for downstream stages.
Barometric compensation
Correction
Removal of atmospheric-pressure variation from a vented / absolute transducer reading. The tiered logic prefers a collocated baro sensor, falls back to the nearest station, and otherwise flags the record BARO_UNCORRECTED.
Datum normalisation
Correction
Application of the per-site datum offset to convert operator-native depth values into the common ground-referenced frame. If site metadata is missing, DATUM_UNVERIFIED is surfaced rather than silently propagated.
Temporal resampling
Correction
Alignment of irregular operator cadences (daily → quarterly) onto a shared time grid, preserving each record's data_age_hours so freshness can be surfaced at delivery.
Quality flags
VALID
Flag
Record passed all corrections and freshness checks. Safe for primary downstream use in the MIS.
STALE
Flag
Triggered when data_age_hours exceeds the operator-specific freshness threshold. Record retained, but consumers should de-weight it in trend analysis.
BARO_UNCORRECTED
Flag
Applied when no barometric reference — neither collocated nor nearest-station — is available for a pressure reading. The raw value is preserved so corrections can be applied retrospectively.
DATUM_UNVERIFIED
Flag
Surfaced when per-site datum metadata is missing or inconsistent. The recorded offset remains in site metadata; the flag makes the gap visible rather than silently absorbed.
Downstream
Puntland MIS
Consumer
The management-information system receiving normalised records from the connector. Consumes the common-vocabulary field water_level_mbgl alongside each record's quality flags.
data_age_hours
Field
Elapsed hours since the logger captured the observation. Computed at ingestion from the record timestamp; drives the STALE flag at delivery time.
